## 2026-04-05 - [Add visible focus states for keyboard accessibility]
**Learning:** This app uses custom interactive elements (like the `.founder-card` divs with `role="button"` and `tabindex="0"`) that completely lacked visual feedback when receiving keyboard focus. Relying on default browser focus outlines is often insufficient, especially on custom dark-themed components where the contrast of the default outline is poor.
**Action:** Always add explicit `:focus-visible` styles with a high-contrast brand color (like `--orange` used here) and appropriate `outline-offset` to ensure all interactive elements, both native and custom, are clearly identifiable when navigating via keyboard. Adjusted `border-radius` of the focus ring to match the shape of specific components (e.g., 50% for circular buttons).
